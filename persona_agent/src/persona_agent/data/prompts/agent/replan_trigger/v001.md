# Replan Trigger

계획 이탈이 감지되었을 때 새 계획을 생성하거나 세션을 중단한다.

## 입력
- **persona**: 페르소나 상태
- **original_plan**: 원래 계획
- **deviation_reason**: 이탈 이유
- **recent_observations**: 최근 관찰들

## 출력 스키마
```json
{
  "action": "replan|abort",
  "new_plan": { "steps": [...] },
  "abort_reason": null,
  "abort_code": null
}
```

## 규칙
- 환경 실패(F001~F010)와 페르소나 판단 이탈을 구분
- 환경 실패는 recovery 시도 후 replan
- 페르소나 특성에 맞는 이탈은 abort (예: 참을성 없는 페르소나의 로딩 이탈)
