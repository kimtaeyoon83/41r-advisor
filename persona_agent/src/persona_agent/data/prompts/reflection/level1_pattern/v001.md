# Level 1 Reflection — 패턴 추출

이 페르소나가 최근 N회의 세션에서 수집한 observation 들을 읽고,
**반복되는 행동 패턴 2~3개**를 1인칭 서술로 추출한다.

## 입력
- `persona_soul`: 페르소나 정체성 (5축 traits + 서술)
- `observations`: [{obs_id, timestamp, content, page_state, action, reason}, ...]
- `existing_reflections`: 이미 있는 level1 reflection 요약 (중복 방지용)

## 출력 스키마
```json
{
  "patterns": [
    {
      "text": "나는 pricing 페이지를 만나면 비교표를 먼저 스캔하고, 'Contact Sales' 표기가 보이면 즉시 이탈하는 경향이 있다.",
      "evidence_obs_ids": ["o_abc123", "o_def456", "o_789"],
      "confidence": "high"
    }
  ],
  "summary": "이 페르소나의 최근 행동을 한 문장으로"
}
```

## 규칙
1. **1인칭 서술** — 페르소나의 내러티브에 주입될 것이므로 "나는 …"로 시작
2. **증거 기반** — 각 pattern은 최소 2개 이상의 obs_id를 참조해야 함 (단발성 사건 금지)
3. **soul과의 불일치 허용** — 만약 observation이 soul의 trait과 다른 방향을 가리키면 그 사실을 지적
   - 예: "내 soul은 research_depth=0.5지만, 실제 세션에서는 평균 7턴 이상을 소비해 research=0.7 이상 수준으로 행동했다"
4. **패턴은 2~3개로 제한** — 더 많으면 노이즈
5. **confidence**: observations 수 (`high` ≥ 5, `medium` 3~4, `low` 2)
6. 기존 reflection과 겹치지 않게 — `existing_reflections`에 비슷한 주제가 있으면 **새로운 각도만** 추출
7. 새로운 패턴이 없으면 `{"patterns": [], "summary": "기존 reflection에서 크게 벗어난 신규 패턴 없음"}` 반환 OK

## 메타 (soul drift 감지)
출력 `patterns` 중 **soul의 trait 값과 반대 방향** 행동이 있다면 `meta` 필드 추가:
```json
"meta": {
  "soul_drift_detected": true,
  "trait": "research_depth",
  "soul_value": 0.5,
  "observed_value": 0.75,
  "reason": "최근 5 세션에서 평균 8턴, 깊이 있는 비교 기사 독해 동작 반복"
}
```
이 필드는 추후 L3 (soul revision) 트리거에 사용된다. 단일 세션 기반으로는 추출 금지 — 3 세션 이상 누적된 경우만.
