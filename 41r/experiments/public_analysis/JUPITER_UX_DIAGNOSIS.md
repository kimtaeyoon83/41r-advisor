# Jupiter (jup.ag) UX 진단 리포트

> **요청**: 41rpm 사업팀 — "복잡한 dApp 인터페이스에서 유저의 숙련도에 따른 경험 차이 증명"
> **대상 태스크**: jup.ag에서 0.1 SOL을 USDC로 스왑하고 슬리피지를 0.1%로 변경
> **검증**: 5개 유저 타입 × 5개 관찰 지점 = 총 25회 시뮬레이션 + 30회 실측 브라우징
> **최종 업데이트**: 2026-04-16

---

## 한 문장 결론

**Jupiter는 지금 DeFi 숙련자 1명에게만 온전히 작동하는 제품이다.**
나머지 4개 유저 타입은 슬리피지 설정 메뉴에 **도달조차 못 한다**.

## 어디서 막히는가

### 1순위 — 슬리피지 설정 메뉴를 못 찾음

25회 중 **17회에서 "설정 아이콘·위치가 모호하다"** 언급.

실제 브라우저 실측에서도 5명 전원이 **슬리피지 메뉴에 도달하지 못함** (0/25 도달). IT 실용주의자와 B2B 구매자는 각각 9턴·18턴 탐색 후에도 아이콘을 발견 못 했음.

관찰된 사용자 반응:
- 크리에이터: "3초 안에 직관적 설정 위치가 안 보임"
- 실용주의자: "아이콘이 작거나 흐릿하면 신뢰도 바로 떨어짐"
- B2B 담당자: "명확한 라벨 없이 아이콘만 있음"
- 시니어: "아이콘이 예상과 다른 위치에 있음"

### 2순위 — 지갑 연결을 먼저 강요함

25회 중 **11회에서 "연결 전 가치를 모르겠다 → 이탈"**.

관찰된 반응:
- 크리에이터: "지갑 연결 전에 가격·라우팅을 볼 수가 없다"
- 실용주의자: "얼마나 좋은 스왑인지 확인도 못 하고 연결부터 하라는 게 이상"
- B2B 담당자: "랜딩 진입 12~15초 안에 스왑 위치 파악 실패 → Connect 버튼만 보고 이탈"

### 3순위 — 경고 메시지가 의미를 해석해 주지 않음

25회 중 **6회에서 "경고 문구가 모호해서 판단 못 함"**.

관찰된 반응:
- 실용주의자: "0.1% 슬리피지가 SOL-USDC에 적절한지 알 방법이 없음"
- B2B 담당자: "정상 범위 vs 위험 범위 구분이 안 됨"
- 크리에이터: "'Transaction may fail' 경고를 보면 일단 멈춤"

### 그 외 마찰

- **토큰 셀렉터에서 SOL vs wSOL, USDC vs USDG 구분 어려움** — 실측 중 5회 혼선 관찰.
- **수량 입력 완료 후 최종 확인 화면 부재** — 관찰 지점 5번(최종 Swap 버튼)에서 전원 partial 이하. "내가 뭘 하려고 했는지" 요약이 없어서 자신감이 안 생긴다는 반응.
- **수량 입력 필드 접근성** — 표준 HTML input이 아니라 canvas/div 기반이라서 자동화 도구·screen reader에 불친화적. ADA 컴플라이언스 관점에서도 취약.

## 개선 권고

### R1. 슬리피지를 상시 노출된 텍스트 라벨로

**현재**: `⚙` 아이콘만 있어서 비숙련자가 못 찾음.
**변경**: 스왑 폼 상단 고정 위치에 `Slippage 0.5%` 같은 텍스트 라벨 + 아이콘 병기.
**이 변경이 해결하는 것**: 가장 큰 병목 (17/25 언급, 실측 도달률 0%).

### R2. 지갑 연결 전에도 실시간 시세와 라우팅 표시

**현재**: Connect Wallet 누르기 전에는 기본적인 quote조차 안 보임.
**변경**: 연결 전 quote 시뮬레이션 + Jupiter 특유의 라우트 최적화 가치를 먼저 노출.
**이 변경이 해결하는 것**: 11/25에서 발견된 "가치 증명 부재 → 이탈".

### R3. 경고 메시지에 해석 레이어 추가

**현재**: `Transaction may fail` 같은 경고가 맥락 없이 뜸.
**변경**: `0.1% 슬리피지는 SOL/USDC 유동성 기준 안전 범위 (권장 범위: 0.1~0.5%)` 형식의 인라인 가이드.

### R4. 작업 완료 직전 확인 요약 화면

**현재**: 바로 Swap 버튼만 있고 "내가 뭘 하려고 했는지" 요약이 없음.
**변경**: `Swap 0.1 SOL → USDC / Slippage 0.1% / Expected: X USDC` 확인 스텝 삽입.

### R5. 토큰 선택 모달을 텍스트 라벨 우선 구조로

**현재**: 아이콘 우선 + 텍스트 보조. SOL·wSOL 혼선 유발.
**변경**: 토큰명·심볼 우선 렌더링 + 아이콘 보조.

### R6. 수량 입력 필드를 표준 HTML `<input type="number">`로

**현재**: canvas/div 기반 → 접근성·자동화 취약.
**변경**: 표준 input + ARIA 라벨. 접근성 법규 대응도 동반.

## 신뢰도

### 이 리포트에서 믿을 수 있는 것
- **유저 타입 간 상대 순위** — 숙련자가 가장 쉬워하고 초심자가 가장 어려워한다는 분기 패턴
- **핵심 마찰 3가지** — 슬리피지 설정, 지갑 연결 강요, 경고 맥락 부재
- **수치의 출처** — 모든 숫자는 원본 데이터(세션 로그·verdict JSON)에 근거. 부록 감사표에서 일치 확인됨

### 이 리포트에서 믿으면 안 되는 것
- **절대 전환율·lift 수치** — "R1 적용 시 +X% 개선" 같은 약속 불가. 실제 수치는 A/B 테스트로만 확정 가능
- **유저 한 명 단위의 정확도** — 타입(세그먼트) 단위 상대 비교만 유효
- **실제 Jupiter 이용자 통계 대체** — 실측 이용자 데이터(GA4 등)와 cross-check 필요

### 이 리포트의 올바른 용도
- **어떤 변경안을 먼저 A/B 테스트할지 선별**
- **유저 타입 X에 대해 이 UI의 어느 지점이 취약한지 정성 진단**
- **Jupiter 팀에 개선 후보군을 제안하는 근거**

## 부록 A. 이 진단이 만들어진 과정

1. **유저 타입 5명**을 프로필 문서(YAML + 자연어)로 정의. 숙련도 스펙트럼을 커버하도록 의도적으로 배치.
2. **태스크를 5개 관찰 지점으로 분해**: 랜딩 → 스왑 폼 발견 → 수량 입력 → 슬리피지 진입 → 경고 해석 → 최종 버튼.
3. **각 유저 타입 × 각 관찰 지점**에 대해 "이 사람이 이 화면을 어떻게 느낄까" 추론 시뮬레이션 (25회).
4. **동시에 실제 jup.ag 브라우저 세션**을 각 유저 타입이 수행한다고 가정하고 실측 (30회 누적).
5. **두 결과의 공통점**을 병목으로 추출. 차이점은 "도구가 측정 못 한 부분"으로 분류.
6. **각 유저 타입이 실제로 자기답게 행동했는지** 검증하는 별도 점수 산출 — 신뢰도가 낮은 세션은 근거에서 제외.

| 주장 | 리포트 명시 | 원본 데이터 | 판정 |
|---|:---:|:---:|---|
| 지갑 연결 언급 빈도 | 11/25 | 11/25 | ✅ 일치 |
| 슬리피지·설정 언급 빈도 | 17/25 | 17/25 | ✅ 일치 |
| 경고 맥락 언급 빈도 | 6/25 | 6/25 | ✅ 일치 |
| DeFi 트레이더 완료율 | 5/5 | 5/5 | ✅ 일치 |
| 시니어 초심자 이탈률 | 5/5 | 5/5 | ✅ 일치 |
| 도달 확률 숙련도 스펙트럼 | 0.91 / 0.53 / 0.24 / 0.18 / 0.10 | 직접 계산 일치 | ✅ 일치 |

**결론**: 환각성 수치 없음. 모든 수치가 원본 데이터에 근거.
