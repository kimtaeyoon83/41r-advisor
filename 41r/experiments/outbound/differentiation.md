# 41R vs 기존 도구 — 차별점

> **용도**: 아웃바운드 영업, 투자자 미팅, 내부 포지셔닝
> **업데이트**: 2026-04-14 (Ablation Study 결과 반영)

---

## 한 줄 요약

기존 도구는 **"B가 이겼다"**를 알려줌.
Demographic 모델은 **"일반 유저가 어떻게 반응할지"**를 알려줌.
**41R은 "이 변경에서 세그먼트가 갈라지는가, 왜 갈라지는가"를 진단함.**

## 3층 비교

| 레이어 | 제품 | 답하는 질문 |
|---|---|---|
| **Post-hoc** | Hotjar / Optimizely | "유저가 뭘 했는가" (이미 일어난 일) |
| **Demographic Survey** | 인구통계 AI 패널 | "평균 유저가 어떻게 반응할까" |
| **Individual Persona (41R)** | 41R | "세그먼트 간 반응이 어떻게 갈라질까, 왜" |

각 레이어는 **대체가 아닌 보완**. 41R은 가장 앞단(설계 직전) 레이어.

---

## Ablation Study로 증명된 차별점

Demographic-only baseline 대비 41R(성향 프로필 추가)의 **marginal value** 측정:

| 메트릭 | Demo-only | 41R | 차이 | 통계적 유의성 |
|---|---|---|---|---|
| **이유 구체성 (blinded 1~5)** | 2.75 | **4.83** | **+2.08** | **p=0.00024** ✅ |
| **세그먼트 분기 탐지** | 0.144 | **0.316** | **2.2×** | p=0.14 (n=12 한계) |
| **예측 엔트로피** | 0.282 | **0.604** | **2.14×** | p=0.072 (marginally) |
| **반직관 케이스 분기 탐지** | 0.154 | **0.857** | **5.5×** | - (n=2) |
| A/B 승자 정확도 | 91.7% | 83.3% | -8.4%p | p=1.00 (무의미) |

**결정적 결과**: Blinded LLM 평가자가 arm을 모른 채 12/12 모든 케이스에서 41R 이유를 더 구체적·액션가능하다고 판정.

## 41R이 **유일하게** 답하는 3가지 질문

### 1. "이 변경은 세그먼트별로 다르게 작용할까?"
- Hotjar: 아직 배포 전이라 모름
- Optimizely: 아직 실험 전이라 모름
- Demo AI Panel: "일반 유저는 좋아합니다" (평균만)
- **41R**: "충동형은 +15%, 신중형은 -5% 예상. 분기 점수 0.7 → 세그먼트별 variant 필요"

### 2. "유저가 왜 이탈할까?"
- Hotjar: "여기서 이탈했다" (why 없음)
- UserTesting: 유저가 말해줘야 함 (비용·시간)
- Demo AI Panel: "답답해서" (일반론)
- **41R**: "신중형은 Privacy Policy를 찾으려 Footer까지 4회 스크롤했으나 미도달. 신뢰 검증 단계에서 구조적 차단"

### 3. "A/B 테스트해야 할 만한 변경인가, 아닌가?"
- 기존: 전부 테스트 → 트래픽·시간 낭비
- **41R**: 분기 점수로 사전 스크리닝. 낮으면 skip, 높으면 세그먼트별 설계

---

## 포지셔닝

**41R ≠ A/B 테스트 대체**
**41R ≠ 유저 리서치 대체**
**41R = A/B 테스트 설계 "전" 레이어**

```
[PM 아이디어 10개]
    ↓ Demographic AI Panel (평균 반응 확인, ~$10/개)
[그럴듯한 7개]
    ↓ 41R 세그먼트 진단 (분기 점수 확인, ~$5/개)
[세그먼트 변수까지 설계된 3개]
    ↓ Optimizely A/B 테스트 (실제 유저, 2~4주)
[승자 확정]
    ↓ Hotjar 사후 모니터링
[성과 추적]
```

각 단계의 효율을 3~5배 증대. 기존 도구 대체가 아니라 **앞단에 새 레이어** 추가.

## 세그먼트 분기 점수 — 실무 쓰임

| 분기 점수 | 의미 | 권고 액션 |
|---|---|---|
| **< 0.2** | 전 세그먼트 동일 반응 | 단일 A/B 테스트로 OK |
| **0.2 ~ 0.5** | 일부 세그먼트 차이 | A/B + 세그먼트 분석 |
| **> 0.5** | 강한 세그먼트 분기 | **세그먼트별 variant 설계 필수** |

"B가 이긴다"는 답보다 "세그먼트별로 다르게 반응한다"는 진단이 **CPO 의사결정에 10배 더 유용**.

## 검증 방법론 요약

- 공개 A/B 테스트 30건 역검증 (VWO, HubSpot, Optimizely 등)
- Ablation: Demo-only vs 41R 2-arm 비교
- Blinded 평가: 제3자 LLM이 arm 모른 채 점수
- 데이터 오염 통제: 라벨 셔플 (seed=42), 회사명 제거, 사전 지식 금지
- 통계 검증: Fisher's Exact, paired t-test, sign test

모든 결과 재현 가능: `experiments/ablation/` + `experiments/ab_validation/`

## 한계와 정직함

- **정확도 우위 주장 안 함**: Demo-only가 A/B 승자 예측에선 +8.4%p 우위
- **n=12 케이스 한계**: 본 결과는 방향성 확인. n=30+ 확장 예정
- **텍스트 기반 ablation**: 실제 브라우저 세션 기반 확장 검증 필요
- **H2 타깃**: 절대 수치(전환율 X%) 예측은 실제 GA 데이터 대조로 별도 검증 필요
