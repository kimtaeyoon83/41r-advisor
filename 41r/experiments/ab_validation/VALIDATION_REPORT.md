## 데이터 오염 통제

공개 A/B 케이스는 LLM 학습 데이터에 포함되었을 가능성이 있음. 이를 통제하기 위해:

1. **라벨 셔플**: 12건 중 10건의 A/B 라벨을 무작위로 뒤집음 (seed=42, 재현 가능)
   - 원본(B 항상 승자) 정확도: 100% → 학습 데이터 편향 확인
   - 셔플 후 정확도: **80%** → 라벨에 무관하게 variant 내용을 보고 판단
2. **회사명 제거**: 프롬프트에서 회사명을 제외하여 유명 사례 직접 매칭 차단
3. **사전 지식 사용 금지 지시**: 시스템 프롬프트에 명시
4. **Cache OFF 모드**: `cache_disabled()` 컨텍스트로 감쌈 (Constitution §6.2 준수). 실제 `provider_router.call` 경로는 plan/page/tool 캐시를 사용하지 않으나, 엄밀성을 위해 명시적으로 비활성화

## 틀린 케이스 분석

### 1. Groove — 가격 페이지 레이아웃 (상세 비교 vs 단순화)
- **예측**: 상세 비교 테이블 (3/5 페르소나 선택)
- **실제**: 단순화된 단일 플랜 강조가 +25% 승리
- **오답 원인**: "선택의 역설" — 정보가 많다고 전환이 높지 않음. 신중형·가격민감형·실용주의형이 비교표를 선호했지만, 실제 유저는 비교 피로도로 이탈
- **세그먼트 인사이트는 유효**: 충동형과 시니어는 단순한 쪽을 정확히 선택

### 2. EA SimCity — 체크아웃 할인 배너 유무
- **예측**: 10:10 동점 → 할인 쪽으로 판정
- **실제**: 할인 배너 제거가 +43.4% 승리
- **오답 원인**: 반직관적 결과. 할인 배너가 "더 좋은 딜을 찾으러 이탈" 유발. 충동형+가격민감형이 할인에 반응했지만, 전체 유저 풀에서는 이탈 효과가 더 컸음
- **세그먼트 인사이트는 유효**: 신중형·실용주의형·시니어가 "깔끔한 체크아웃" 정확 선택

### 공통 패턴
두 오답 모두:
- CRO 전문가도 자주 틀리는 반직관적 케이스
- **세그먼트별 예측은 논리적으로 타당** — 집계 방법의 한계

## 세그먼트 분기 감지

12건 중 9건(75%)에서 페르소나 간 의견 분기 감지.

이는 기존 A/B 테스트 도구가 제공하지 못하는 핵심 가치:
- "B가 이겼다"가 아니라 **"충동형은 A를 선호하고, 신중형은 B를 선호한다"**
- 세그먼트별 다른 UX를 설계해야 하는지 판단 근거 제공

## 한계 및 다음 단계

### 현재 한계
1. **공개 케이스만 사용**: LLM 학습 데이터 완전 배제 불가 (셔플로 통제했으나 한계 있음)
2. **텍스트 기반 평가**: 실제 브라우저 세션이 아닌 variant 설명으로 판단
3. **소규모 데이터셋**: 12건으로는 통계적 유의성 주장 어려움 (95% CI: 52~98%)
4. **반직관적 결과에 약함**: 집계 모델이 "상식적 판단"에 편향

### H2 개선 계획
1. **비공개 A/B 케이스 확보**: 파트너사의 실제 테스트 데이터로 검증 (학습 데이터 완전 배제)
2. **브라우저 세션 기반 역검증**: 실제 variant 페이지를 브라우저로 탐색하여 판단 (현재보다 리치한 데이터)
3. **집계 모델 개선**: 세그먼트 크기 가중치, 반직관적 패턴 보정
4. **데이터셋 확장**: 30건+ 으로 통계적 유의성 확보
