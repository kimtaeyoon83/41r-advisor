## 변경 핵심

### Before — Constitution v1.0 (§1)

```
시스템 본질
- 입력: 고객사의 제품 URL(들) + 태스크 + 타겟 유저 세그먼트(자연어)
- 출력: 캘리브레이션된 페르소나가 실제 브라우저에서 제품을 사용한 결과를 비교한 예측 리포트
- 핵심 사용 시나리오: 출시 전 예측 A/B 테스트 (하루 내 결과, 기존 A/B 대비 2~4주 단축)
```

### After — 피벗 후 (Sprint revision 시 반영 예정)

```
시스템 본질
- 입력: 고객사의 제품 URL(들) + 태스크 + 타겟 유저 세그먼트(자연어)
- 출력: AI 페르소나 코호트가 사이트를 사용한 후 "세그먼트별로 다르게 반응할 가능성"을 진단하는 리포트
- 핵심 사용 시나리오: A/B 테스트 설계 전 단계 — PM의 변경 아이디어 N개 중,
  세그먼트별로 갈리는 후보를 식별 (24시간 · 건당 ~$5)
```

### 왜 변경

**검증 데이터 (n=200 Ablation)**:
- 41R 정확도 (A/B 승자 예측): 55% vs Demo 61% — 무의미한 차이 (p=0.073)
- 41R 분기 탐지: 58.5% vs Demo 42.5% — **+16%p, p=0.000009** (매우 유의)

→ "예측 A/B 테스트"라는 원래 포지션은 데이터로 입증 안 됨.
→ "세그먼트 분기 진단"이 통계적으로 입증된 marginal value.

## §11 가설 검증 매핑 변경

### Before

```
H1 바이어 | Sample report + 공개 A/B 역검증 3건 → CPO 30명 콜드 아웃바운드 | 유료 전환율 < 5%
```

### After

```
H1 바이어 v2 | EXECUTIVE_REPORT 발송 → CPO/Head of Growth 30명 | 단계화된 5단계 cut-off
```

단계화 (Kill Criteria):
1. 발송 → 응답 (Kill: <3/30)
2. 응답 → 진단 샘플 요청 (Kill: 0/5, **Pass: ≥3**)
3. 샘플 → 미팅 (Kill: 0/3, Pass: ≥2)
4. 미팅 → 유료 의향 (Kill: 0/2, Pass: ≥1)
5. 의향 → 결제 (Kill: $5+ 0건, Pass: 1건+)

각 단계가 빠른 중간 신호. 이전 단일 임계 ("유료 전환율 5%")은 신호 너무 늦음.
