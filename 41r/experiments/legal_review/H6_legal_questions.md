# H6 규제 검토 — 법률 자문 질문 리스트

> **작성일**: 2026-04-14
> **목적**: H1 검증 전후로 필요한 법률 리스크 파악
> **대응 Constitution**: §11 가설 검증 매핑 (H6), §12 의도적으로 하지 않을 것
> **상태**: 자문 의뢰 전, 내부 정리

---

## 배경

41R은 "AI 페르소나가 실제 브라우저에서 제품을 사용하고, 세그먼트별 행동을 비교한 예측 리포트"를 판매. 다음 영역에서 법률 리스크 존재 가능:

1. FTC/광고 규제 (synthetic reviews)
2. 개인정보 (GDPR, CCPA, k-anonymity)
3. 크롤링 / 브라우저 자동화 (CFAA, robots.txt, ToS)
4. 저작권 (스크린샷, 사이트 콘텐츠)
5. provenance 온체인 기록의 법적 효력

---

## 질문 1. "Synthetic Reviews" FTC 위반 여부

**배경**: FTC는 2024-08 "Rule on the Use of Consumer Reviews and Testimonials"을 통해 AI 생성 리뷰/추천을 금지. Constitution §12에도 "synthetic reviews" 워딩 사용 금지 명시.

**질문**:
1. 41R 리포트가 "페르소나가 느낀 점"을 텍스트로 기술할 때, 이것이 FTC Rule의 "fake review"에 해당하는가?
2. 다음 워딩 중 안전한 것은?
   - ❌ "가상 유저 300명이 좋다고 답했습니다"
   - ✅ "AI 페르소나 시뮬레이션 결과, 충동형 세그먼트의 85%가 CTA 클릭 단계에 도달"
   - ? "예측 모델 기반 전환율 추정"
3. B2B 판매 시(고객이 리포트를 내부용으로만 사용) FTC Rule 적용 범위는?
4. 고객이 리포트를 마케팅 자료로 재사용하면 책임은 누구에게?

**현재 대응**: Constitution §12 워딩 금지 목록 유지. 리포트 프레이밍을 "UX 예측"으로 통일.

---

## 질문 2. 개인정보 / k-anonymity

**배경**: 페르소나 정의 파일(`soul/v002.md`)에 "35세 서울 서초 회계법인 시니어" 같은 식별 가능 속성 포함. 실제 인물 매칭 가능성 존재.

**질문**:
1. 페르소나의 연령+지역+직업 조합이 k-anonymity(k=5 기준) 위반 가능성이 있는가?
2. 만약 실제 유저와 일치하는 프로필이 존재한다면, 해당 유저가 "내 개인정보로 학습했다"는 클레임을 걸 수 있는가?
3. 페르소나 생성에 사용한 참고 데이터(Baymard, NNGroup 등)의 2차 이용 허용 범위는?
4. GDPR/CCPA 대상 EU/캘리포니아 고객 판매 시 추가 고지 의무는?

**현재 대응**: 페르소나는 실제 인물 기반 아닌 원형 모델. 식별 속성을 4개 이하로 유지. 단, 문서화 필요.

## 질문 3. 브라우저 자동화 / 크롤링

**배경**: M2 Browser Runner가 Stagehand + Chromium으로 실제 사이트를 탐색. 41R이 고객사 외 사이트(예: 경쟁사)도 분석 가능.

**질문**:
1. 고객이 "경쟁사 사이트 분석"을 요청하여 41R이 제3자 사이트를 크롤링한 결과를 리포트로 제공할 때:
   - CFAA(Computer Fraud and Abuse Act) 위반 가능성?
   - 해당 사이트 ToS 위반 시 41R 책임 vs 고객 책임?
2. robots.txt 준수 의무는?
3. Rate limiting, 로그인 필요 페이지 접근 정책은?
4. 2023 *hiQ Labs v. LinkedIn* 판례의 적용 범위 (공개 데이터 스크레이핑)?
5. Stagehand 라이선스(MIT 기반)가 B2B 상용 서비스에 주는 제약?

**현재 대응**: H1에서는 고객사 자체 사이트만 분석한다는 전제. 경쟁사 분석은 H2에서 재검토.

## 질문 4. 스크린샷 / 사이트 콘텐츠 저작권

**배경**: M2가 사이트 스크린샷 캡처, 리포트에 일부 포함. 사이트 UI/로고/텍스트는 타사 저작권 대상.

**질문**:
1. 리포트에 고객사 사이트 스크린샷 포함이 fair use에 해당하는가? (UX 비평/예측 목적)
2. 경쟁사 스크린샷을 비교 자료로 사용할 때 제약은?
3. 리포트를 고객사 외부로 공유(예: 투자자 IR) 허용 조건?
4. 스크린샷 내에 표시된 써드파티 광고/임베디드 콘텐츠의 처리?

**현재 대응**: 리포트 내부용 전제. 외부 공유 시 고객사가 자체 권리 확인 의무를 계약서에 명시 필요.

## 질문 5. Provenance / SAS 온체인 기록

**배경**: Constitution §1, §4 — 모든 관찰을 SAS (Stellar/Solana 등) 온체인에 기록. M4 Provenance (H1에서 stub).

**질문**:
1. 온체인 기록된 observation(예: "유저 X의 행동 패턴")이 GDPR "right to erasure"(제17조)와 충돌?
2. 블록체인 기록의 "법적 증거 효력" 인정 국가 범위?
3. 고객사가 온체인 기록 중단/삭제를 요청할 경우 기술적/법적 대응?
4. 한국 개인정보보호법(PIPA) 상 "파기 불가능한 기록" 허용 조건?

**현재 대응**: H1에선 stub. H2 전에 결정. 최악의 경우 온체인 대신 append-only 파일 시스템으로 대체.

## 질문 6. 정확도 주장 / 면책

**배경**: 검증 리포트에 "83.3% A/B 예측 정확도" 수치 표시. Constitution §12 — "페르소나 개별 정확도 주장 금지".

**질문**:
1. "AI 예측 정확도 83.3%" 마케팅 표기의 법적 리스크?
2. 고객이 리포트 기반 의사결정으로 손실 시 책임 범위?
3. 필수 면책 조항 템플릿(contract clause) 권장 사항?
4. 한국 전자상거래법상 "AI 서비스 고지 의무" 적용 여부?

**현재 대응**: 리포트에 "실제 유저 테스트 대체 아님" 명시. 단, 계약서 템플릿 필요.
